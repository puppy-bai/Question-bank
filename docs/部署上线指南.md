# 微信题库小程序部署上线指南

版本：v0.1  
日期：2026-05-25

## 1. 上线链路总览

```text
注册小程序账号
  -> 获取 AppID
  -> 安装微信开发者工具
  -> 初始化项目
  -> 开通云开发环境
  -> 开发和真机预览
  -> 上传体验版
  -> 小程序后台提交审核
  -> 审核通过后发布
  -> 运营监控和版本迭代
```

## 2. 账号准备

### 2.1 注册小程序

需要在微信公众平台注册小程序账号，并完成主体信息。个人主体和企业主体可用能力不同，涉及品牌、服务类目、备案和审核材料时，学校或组织主体通常更稳。

需要准备：

- 小程序名称。
- 主体类型：学校、企业、组织、个体工商户或个人。
- 管理员微信。
- 邮箱。
- 小程序头像、简介、服务类目。
- 如涉及学校或机构服务，准备相关资质。

### 2.2 获取 AppID

注册完成后，在微信公众平台后台获取 AppID。开发时需要在微信开发者工具中填入 AppID。

## 3. 本地开发环境

需要安装：

- 微信开发者工具。
- Node.js。
- Git。

开发者工具用于本地调试、真机预览、上传体验版和代码上传。后续如果要做自动化上传，可以接入 `miniprogram-ci`。

## 4. 后端部署方案

### 4.1 推荐：微信云开发 / CloudBase

第一版推荐使用微信云开发：

- 不需要单独购买服务器。
- 不需要先处理自有 API 域名和 HTTPS 证书。
- 小程序可以直接调用云函数、云数据库和云存储。
- 后台敏感逻辑可以放在云函数中。

基础资源：

- 云数据库：题库、题目、练习记录、错题、收藏。
- 云函数：登录、判题、Word 导入、老师数据统计、管理员操作。
- 云存储：题图、Word 导入文件、解析图片。

### 4.2 备选：自建后端

如果后期需要复杂后台、跨平台、强报表或更灵活的数据库，可以使用自建后端：

- 腾讯云轻量服务器 / 云服务器。
- Node.js / Java / Python 后端。
- MySQL / PostgreSQL / MongoDB。
- HTTPS 域名。
- 在小程序后台配置 request 合法域名。

这条路线更灵活，但上线和运维成本更高。

## 5. 云开发环境配置

1. 在微信开发者工具中打开项目。
2. 开通云开发。
3. 创建环境，例如：
   - `question-bank-dev`
   - `question-bank-prod`
4. 在小程序初始化时配置环境 ID。
5. 创建数据库集合。
6. 配置集合权限。
7. 部署云函数。

建议至少区分开发环境和生产环境，避免测试数据污染正式用户。

## 6. 数据库权限建议

第一版建议：

- `question_banks`、`chapters`、`questions`：普通用户只读已发布内容。
- `attempts`、`wrong_questions`、`favorites`、`study_sessions`：学生只读写自己的数据。
- 老师数据查询必须通过云函数，按班级负责关系过滤。
- 管理相关集合和操作：仅云函数可写。
- Word 批量导入、上下架、角色配置、题库可见范围配置：必须校验管理员密码状态、管理员 OpenID 和管理员角色。

## 7. 题库可见范围准备

第一版不做收费和支付，只做校内可见范围控制：

1. 管理员可以设置题库对指定班级、学生或老师可见。
2. 学生只能看到自己班级或账号范围内可见的题库。
3. 老师只能查看自己负责班级或学生的数据。
4. 可见范围变更需要写入管理日志。

支付、会员、收费题库放入二期需求，等第一版刷题闭环和校内使用跑通后再单独设计。

## 8. 小程序上传与发布

### 8.1 开发调试

1. 在微信开发者工具中打开项目。
2. 本地模拟器调试。
3. 使用真机预览验证登录、网络、云函数、页面兼容。

### 8.2 上传体验版

1. 在微信开发者工具中点击上传。
2. 填写版本号和项目备注。
3. 到微信公众平台后台查看体验版本。
4. 添加体验成员进行测试。

### 8.3 提交审核

1. 确认服务类目。
2. 补齐隐私政策和用户协议。
3. 准备功能说明和测试账号。
4. 提交审核。
5. 根据审核反馈修改。

### 8.4 发布上线

审核通过后，在微信公众平台后台发布。发布后建议观察：

- 云函数错误日志。
- 用户访问量。
- 题目加载速度。
- 答题提交失败率。
- 用户反馈。

## 9. 自动化发布预留

后期可使用 `miniprogram-ci` 做：

- 自动预览。
- 自动上传体验版。
- CI 环境构建。
- 拉取 sourceMap。

自动化上传需要在微信公众平台配置代码上传密钥和 IP 白名单。

## 10. 合规与审核注意事项

题库小程序通常需要关注：

- 服务类目是否匹配题库内容。
- 是否涉及考试培训、职业资格、教育机构等资质要求。
- 是否收集手机号、微信头像昵称等个人信息。
- 是否需要隐私政策。
- 题库内容是否侵犯版权。
- 老师查看学生数据是否已获得学校内部管理授权。
- 学生身份、班级、答题记录属于个人信息，必须在隐私政策中说明用途。

如果只是学校内部学习且不收费，审核压力通常较低；如果后续商业化收费或涉及特定考试资质，需要提前确认合规边界。
