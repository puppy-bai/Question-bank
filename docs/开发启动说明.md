# 开发启动说明

版本：v0.1  
日期：2026-05-25

## 1. 打开项目

1. 安装微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：`C:\Users\Puppy\Desktop\题库小程序`。
4. AppID 先可使用测试号，正式上线前换成真实小程序 AppID。

## 2. 配置云开发环境

1. 在微信开发者工具中开通云开发。
2. 创建环境，例如：`question-bank-dev`。
3. 把环境 ID 填入：

```js
// miniprogram/config/env.js
module.exports = {
  envId: '你的云开发环境 ID'
};
```

## 5. 首次管理员初始化

第一版支持从隐藏管理入口首次初始化管理员：

1. 打开小程序首页。
2. 连续点击底部版本区域 5 次，进入管理入口。
3. 输入管理员密码，首次使用会自动创建管理员配置，并把当前微信 OpenID 绑定为管理员。
4. 后续再进入管理入口时，需要输入同一个管理员密码。
5. 后续新增管理员时，必须由已登录管理员在“账号与班级管理”中把对应用户角色设置为管理员；仅知道密码但 OpenID 未授权的用户不能进入管理后台。

如果希望首次初始化更安全，可以在 `adminLogin` 云函数环境变量中设置：

```text
ADMIN_SETUP_TOKEN=自定义初始化令牌
```

设置后，首次初始化必须在页面里填写相同令牌。初始化完成后，令牌不再用于日常登录。

## 6. Word 导入流程

1. 管理员登录管理入口。
2. 进入 Word 题库导入。
3. 选择 `.docx` 文件。
4. 小程序上传文件到云存储。
5. `parseWord` 生成导入预览。
6. 输入题库名称。
7. 点击确认导入。
8. `confirmImport` 写入 `question_banks`、`chapters`、`questions` 和 `import_jobs`。

## 7. 学生刷题流程

1. 题库页调用 `getBanks` 获取当前账号可见的已发布题库。
2. 练习页调用 `getQuestions` 获取题目，并在云函数内再次校验题库可见权限。
3. 学生提交答案时调用 `submitAnswer`。
4. 系统写入 `attempts`。
5. 答错时写入或更新 `wrong_questions`。
6. 错题页调用 `getWrongQuestions` 展示未掌握错题。
7. 错题页可跳回对应题库复习，也可调用 `markWrongQuestionMastered` 标记已掌握。
8. 答题页可调用 `toggleFavorite` 收藏或取消收藏题目。
9. 我的页面进入收藏夹，调用 `getFavorites` 查看收藏题目并跳回对应题库复习。

练习页当前支持：

- 答题模式：提交后由云函数判题，再展示答案和解析。
- 背题模式：进入时请求答案，直接展示参考答案和解析。
- 答题卡：查看当前进度、正确/错误状态，并可跳题。
- 题目收藏：收藏状态随题目加载，收藏夹可直接定位到对应题目。
- 错题复习：错题本可跳回对应题目，并支持标记已掌握。
- 本地断点续答：退出后再次进入同一题库会恢复当前题号和本地答题状态。
- 清空本地记录：只清除本机练习进度，不删除云端答题记录和错题。
- 练习结果页：完成后展示总题数、正确数、错误数、未答数、正确率和用时。

## 8. 老师查看学生数据流程

1. 老师工作台调用 `teacherGetDashboard`。
2. 如果当前用户是老师，只读取其负责班级下的学生。
3. 如果当前用户是管理员，可查看全部学生，便于早期调试。
4. 点击学生进入详情页，调用 `teacherGetStudentStats`。
5. 学生详情展示答题次数、正确率、未掌握错题和最近答题记录。

管理员可以通过隐藏管理入口进入“账号与班级管理”，维护：

- 班级列表：写入 `classes`
- 用户角色和姓名：写入 `users`
- 学生分班：写入 `student_classes`，并同步 `users.className`
- 老师负责班级：写入 `teacher_classes`

第一版账号绑定流程建议先保持简单：学生和老师先打开一次小程序，`login` 云函数会自动创建 `users` 记录；管理员随后在管理页中把该 OpenID 改成老师或学生，并配置班级关系。

## 9. 题库可见范围流程

1. 管理员进入隐藏管理入口。
2. 进入“题库可见范围”。
3. 选择题库。
4. 选择“全校可见”或“指定班级”。
5. 保存后写入 `bank_visibility`。
6. `getBanks` 会只返回当前账号可见题库；`getQuestions` 会在进入答题前再次校验权限。

未配置 `bank_visibility` 的题库默认全校可见，方便早期导入后直接测试。

## 10. 题目手动维护流程

1. 管理员进入隐藏管理入口。
2. 进入“题目维护”。
3. 选择题库，页面会加载该题库最近 100 道题。
4. 点击题目可编辑题干、题型、章节、选项、答案、解析和状态。
5. 点击“保存题目”后写入 `questions`，并同步 `question_banks.questionCount`。
6. 可通过列表右侧按钮快速下架或重新发布单题。

当前手动维护以文本题为主；图片题的图片替换和题图上传后续单独补。
