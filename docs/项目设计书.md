# 第三方题库小程序项目设计说明书

版本：v0.2  
日期：2026-06-04  
状态：新版产品设计稿

## 1. 项目定位

本项目是一个第三方题库小程序，核心目标是让管理员可以快速导入、维护和发布题库，让普通用户可以选择题库后进行章节练习、随机测试、模拟考试、错题复习和收藏复习。

小程序不以某个固定组织为中心，而是以“题库内容”和“用户刷题体验”为中心。第一版先完成可实际使用的题库导入、题库管理、章节刷题、测试考试、用户数据统计和基础权限控制。

## 2. 产品目标

### 2.1 第一版目标

- 进入小程序后先出现登录选择界面，分为“管理员登录”和“用户登录”。
- 管理员登录后进入管理后台，可以管理题库、章节、题目、导入 Word、发布/下架/删除题库、查看用户使用数据。
- 用户登录后进入用户首页，可以选择题库、查看章节、进行练习、测试、考试、错题复习、收藏复习和查看个人记录。
- 题库支持按章节组织，用户可以选择整个题库、单个章节或多个章节开始答题。
- 题型支持单选、多选、判断、填空、简答；客观题自动判题，主观题展示参考答案或留作后续人工/AI 批改。
- Word 题库导入支持固定模板，导入前解析预览，确认后写入题库。
- 管理员可以在后台查看题库使用情况、用户答题次数、正确率、错题排行、考试成绩等数据。
- 第一版不做收费、会员、分销和复杂商业化能力，先保证题库和刷题闭环稳定。

### 2.2 后续扩展目标

- 题库商品化、会员、订单和微信支付。
- 多管理员协作和操作权限细分。
- 图片题、公式题、音视频题。
- AI 生成解析、AI 出题、AI 错题诊断。
- H5 后台管理系统。
- 数据导出、排行榜、证书、学习计划。

## 3. 用户角色

### 3.1 管理员

管理员是题库运营和内容维护人员，负责：

- 新增题库。
- 导入 Word 题库。
- 管理章节目录。
- 新增、编辑、删除、上下架题目。
- 设置题库状态和可见范围。
- 配置测试和考试规则。
- 查看用户答题数据、考试成绩和题库使用情况。
- 处理异常题目、反馈和导入错误。

管理员登录需要管理员密码，并绑定当前微信 OpenID。首次初始化后，后续管理员账号由已有管理员添加或授权。

### 3.2 用户

用户是刷题和考试人员，负责：

- 登录小程序。
- 浏览可用题库。
- 选择题库和章节。
- 进行章节练习、随机练习、专项测试、模拟考试。
- 查看答题结果、解析、错题和收藏。
- 继续未完成的练习或考试。
- 查看个人学习数据。

用户不需要复杂注册，第一版采用微信 OpenID 自动建档，可选填写昵称或备注名。

## 4. 总体入口设计

### 4.1 首屏登录选择

用户首次进入小程序后看到登录选择页：

```text
题库小程序

[用户登录]
[管理员登录]
```

用户登录流程：

1. 点击“用户登录”。
2. 调用 `login` 云函数获取 OpenID。
3. 如果用户不存在，自动创建普通用户。
4. 进入用户首页。

管理员登录流程：

1. 点击“管理员登录”。
2. 输入管理员密码。
3. 调用 `adminLogin` 云函数。
4. 校验密码、OpenID 和管理员角色。
5. 进入管理员后台首页。

### 4.2 登录状态

- 本地缓存登录角色和最近入口。
- 管理员登录状态建议设置有效期，例如 24 小时。
- 用户登录状态可以长期保留，但每次关键操作仍以云端 OpenID 为准。
- “我的”页面提供切换入口，方便从用户端进入管理员登录。

## 6. 用户端功能设计

### 6.1 用户首页

首页展示用户最需要继续操作的内容：

- 最近练习的题库。
- 未完成的测试或考试。
- 今日答题数量、正确率、累计刷题数。
- 推荐题库或最新题库。
- 错题待复习数量。

首页操作：

- 继续上次练习。
- 进入题库列表。
- 进入错题本。
- 查看个人记录。

### 6.2 题库列表

题库列表展示管理员已发布、当前用户可见的题库。

题库卡片字段：

- 题库名称。
- 题库简介。
- 章节数量。
- 题目数量。
- 最近更新时间。
- 用户进度。
- 正确率。

筛选能力：

- 全部题库。
- 最近练过。
- 已收藏题库。
- 关键词搜索。
- 分类筛选。

### 6.3 题库详情

用户点击题库后进入题库详情。

题库详情包括：

- 题库名称和简介。
- 总题量、章节数、用户已做题量。
- 章节列表。
- 快捷入口：章节练习、随机练习、模拟测试、考试模式。
- 用户在该题库下的错题数和收藏数。

### 6.4 章节选择

章节是题库的核心结构。

章节展示方式：

```text
第一章 基础知识    120 题  已做 35 题
第二章 专项训练    260 题  已做 80 题
第三章 综合应用    180 题  已做 20 题
```

章节操作：

- 点击单个章节开始练习。
- 勾选多个章节组合练习。
- 查看章节正确率和错题数。
- 只练未做题。
- 只练错题。
- 只练收藏题。

### 6.5 练习模式

练习模式用于日常刷题，强调即时反馈。

规则：

- 用户选择题库或章节后开始。
- 可选择顺序练习或随机练习。
- 客观题提交后立即显示正确/错误、正确答案、解析。
- 判断题和选择题直接点击选项作答。
- 填空题输入答案，第一版可做简单文本匹配。
- 简答题展示参考答案，不强制自动判分。
- 支持上一题、下一题、答题卡、收藏、查看解析。
- 支持本地进度恢复。

适用场景：

- 日常复习。
- 分章节刷题。
- 错题重做。
- 收藏题复习。

### 6.6 测试模式

测试模式用于阶段性自测，介于练习和考试之间。

规则：

- 用户选择题库、章节和题量。
- 系统按规则抽题，例如 20 题、50 题、100 题。
- 答题过程中不立即显示答案。
- 用户完成后统一提交。
- 结果页展示得分、正确率、用时、错题列表和解析。
- 错题自动进入错题本。

建议配置项：

- 题量。
- 题型比例。
- 是否随机。
- 是否限时。
- 是否允许中途退出后继续。

### 6.7 考试模式

考试模式用于正式模拟考试，强调规则和结果记录。

规则：

- 管理员先配置考试。
- 考试可以是固定试卷，也可以是抽题规则。
- 考试开始后显示倒计时。
- 答题过程中不显示答案和解析。
- 到时间自动交卷。
- 交卷后生成考试成绩记录。
- 考试记录可在个人记录和管理员数据统计中查看。

考试配置项：

- 考试名称。
- 关联题库。
- 章节范围。
- 题型比例。
- 总题量。
- 总分。
- 及格线。
- 考试时长。
- 是否允许重复考试。
- 是否显示解析。
- 开放时间和结束时间。

### 6.8 错题本

错题本按用户维度保存。

功能：

- 按题库筛选错题。
- 按章节筛选错题。
- 按题型筛选错题。
- 查看错误次数。
- 查看最近错误时间。
- 重新练习错题。
- 标记已掌握。
- 答对指定次数后自动移出未掌握列表。

错题来源：

- 练习答错。
- 测试答错。
- 考试答错。

### 6.9 收藏夹

收藏夹用于保存用户主动关注的题目。

功能：

- 收藏/取消收藏。
- 按题库查看收藏。
- 按章节查看收藏。
- 收藏题专项练习。

### 6.10 学习记录

学习记录展示用户历史行为。

字段：

- 时间。
- 题库。
- 模式：练习、测试、考试。
- 答题数量。
- 正确数量。
- 正确率。
- 用时。
- 得分。

用途：

- 用户复盘。
- 管理员查看用户使用数据。

## 7. 管理员端功能设计

### 7.1 管理后台首页

管理后台首页展示关键运营数据：

- 题库总数。
- 已发布题库数。
- 总题量。
- 今日答题次数。
- 今日活跃用户。
- 近期考试次数。
- 高频错题排行入口。
- 最近导入任务。

### 7.2 题库管理

题库管理是管理员最核心的功能。

题库字段：

- 题库名称。
- 题库简介。
- 分类。
- 封面图。
- 状态：草稿、导入中、已发布、已下架、已删除。
- 章节数量。
- 题目数量。
- 创建人。
- 更新时间。

题库操作：

- 新增题库。
- 重命名题库。
- 编辑简介和分类。
- 发布题库。
- 下架题库。
- 删除题库。
- 重新统计题量。
- 进入题目管理。
- 进入章节管理。
- 查看题库数据。

删除策略：

- 第一版建议使用软删除，避免误删后无法恢复。
- 已删除题库不在用户端展示。
- 后续可增加“彻底删除”功能，清理题目、记录和文件。

### 7.3 Word 导入

导入流程：

1. 管理员选择 Word 文件。
2. 上传到云存储。
3. 云函数解析 Word。
4. 返回题目预览、章节、题量、错误和警告。
5. 管理员输入题库名称或选择已有题库。
6. 确认导入。
7. 系统分批写入题库、章节和题目。
8. 导入完成后发布或保存为草稿。

导入要求：

- 支持 `.docx`。
- 支持章节识别。
- 支持单选、多选、判断、填空、简答。
- 支持答案和解析。
- 支持导入前预览。
- 支持错误提示。
- 大题库必须分批导入，避免云函数超时。

### 7.4 章节管理

章节管理用于维护题库目录。

功能：

- 查看章节树。
- 新增章节。
- 重命名章节。
- 删除空章节。
- 调整章节顺序。
- 合并章节。
- 将题目移动到其他章节。
- 查看章节题量和正确率。

章节数据建议支持层级：

```text
题库
  ├─ 第一章
  │   ├─ 第一节
  │   └─ 第二节
  └─ 第二章
```

第一版可以先支持一级章节，后续再做多级章节。

### 7.5 题目管理

题目管理功能：

- 按题库筛选。
- 按章节筛选。
- 按题型筛选。
- 按状态筛选。
- 关键词搜索题干。
- 新增题目。
- 编辑题干。
- 编辑选项。
- 编辑答案。
- 编辑解析。
- 修改章节。
- 上架/下架题目。
- 删除题目。
- 批量操作。

题目状态：

- 草稿。
- 已发布。
- 已下架。
- 已删除。

### 7.6 测试/考试配置

管理员可以配置两类活动：

测试配置：

- 适合用户自测。
- 可以由用户自由选择是否参加。
- 可以不限制时间。
- 可重复进行。

考试配置：

- 更正式。
- 有时间限制、分数、及格线。
- 可以记录成绩。
- 可控制开放时间。

配置字段：

- 名称。
- 类型：测试、考试。
- 关联题库。
- 章节范围。
- 抽题方式：固定题目、随机抽题。
- 题量。
- 题型比例。
- 总分。
- 及格线。
- 时长。
- 开放状态。
- 开放时间。
- 结束时间。
- 是否允许重复。
- 结果是否展示答案解析。

### 7.7 用户管理

第一版用户管理保持简单：

- 用户首次登录后自动生成用户记录。
- 管理员可以查看用户列表。
- 管理员可以设置用户备注名。
- 管理员可以禁用异常用户。
- 管理员可以查看单个用户的答题数据。

用户字段：

- OpenID。
- 昵称或备注名。
- 角色：user、admin。
- 状态：normal、disabled。
- 首次登录时间。
- 最近登录时间。
- 累计答题数。
- 平均正确率。

### 7.8 数据统计

管理员数据统计分为四类。

用户统计：

- 用户总数。
- 今日活跃用户。
- 最近 7 天活跃用户。
- 人均答题数。
- 用户正确率排行。

题库统计：

- 题库访问次数。
- 题库答题次数。
- 题库平均正确率。
- 题库完成率。
- 热门题库。

题目统计：

- 高频错题。
- 题目错误率。
- 题目作答次数。
- 长期无人作答题目。

考试统计：

- 考试参与人数。
- 平均分。
- 最高分。
- 最低分。
- 通过率。
- 成绩列表。

## 8. 题型设计

### 8.1 单选题

- 选项使用 A、B、C、D。
- 用户点击一个选项。
- 答案数组保存一个值，例如 `["A"]`。
- 自动判题。

### 8.2 多选题

- 用户可选择多个选项。
- 答案数组保存多个值，例如 `["A", "C"]`。
- 完全一致才判定正确。
- 可选支持少选得部分分，第一版暂不做。

### 8.3 判断题

- 页面直接显示“正确”和“错误”两个选项。
- 用户点击作答。
- 答案保存 `["正确"]` 或 `["错误"]`。
- 自动判题。

### 8.4 填空题

- 用户输入答案。
- 支持多个空用换行或逗号分隔。
- 第一版做简单文本匹配。
- 后续支持同义答案、忽略大小写、忽略空格。

### 8.5 简答题

- 用户输入答案。
- 第一版不自动判分，提交后展示参考答案。
- 后续可扩展人工判分或 AI 辅助判分。

## 9. 核心业务流程

### 9.1 管理员添加题库流程

```text
管理员登录
  -> 题库管理
  -> 新增题库
  -> Word 导入
  -> 解析预览
  -> 确认导入
  -> 章节和题目写入
  -> 检查题目
  -> 发布题库
  -> 用户端可见
```

### 9.2 用户练习流程

```text
用户登录
  -> 题库列表
  -> 题库详情
  -> 选择章节
  -> 选择练习模式
  -> 答题
  -> 即时判题
  -> 查看解析
  -> 完成练习
  -> 查看结果
  -> 错题进入错题本
```

## 12. 权限设计

### 12.1 用户权限

- 只能查看已发布且可见的题库。
- 只能提交自己的答题记录。
- 只能查看自己的错题、收藏和学习记录。
- 不能直接写入题库、章节和题目集合。

### 12.2 管理员权限

- 管理员操作必须通过云函数。
- 云函数必须校验 OpenID 是否具备管理员角色。
- 管理员密码只用于登录入口，不作为唯一权限依据。
- 管理员操作写入 `admin_logs`。

### 12.3 数据库权限建议

- `question_banks`、`chapters`、`questions`：客户端只读已发布内容，写入必须走云函数。
- `attempts`、`wrong_questions`、`favorites`、`practice_sessions`：用户只能读写自己的数据。
- `exam_configs`：用户只读已发布考试，管理员写入走云函数。
- `exam_records`：用户只能读自己的成绩，管理员通过云函数汇总查看。
- `settings`、`admin_logs`：仅云函数访问。

## 16. 验收标准

第一版达到可用状态时，应满足：

- 打开小程序后先进入登录选择页。
- 用户登录后只能看到用户功能。
- 管理员登录后只能进入管理功能。
- 管理员可以新增、导入、发布、删除题库。
- 管理员可以查看题库题量、用户答题数据和考试数据。
- 用户可以选择题库。
- 用户可以选择章节。
- 用户可以进行章节练习。
- 用户可以进行随机测试。
- 用户可以进行考试。
- 判断题和选择题均为点击选项作答。
- 答题记录、错题、收藏可以保存。
- 用户端不出现管理功能。
- 管理员端不依赖客户端直接写数据库。
- 文案不绑定特定组织场景。

## 17. 风险与对策

| 风险 | 对策 |
| --- | --- |
| Word 格式不统一 | 固定模板、导入预览、错误提示 |
| 大题库导入超时 | 分批导入、断点重试、进度显示 |
| 章节识别不准 | 模板规范化，支持导入后手动调整章节 |
| 用户误进管理端 | 管理端需要密码和 OpenID 权限双校验 |
| 题库误删 | 第一版采用软删除 |
| 考试作弊或答案泄露 | 考试模式不提前返回答案，交卷后按配置展示 |
| 数据统计慢 | 关键数据写入会话表，后台统计做聚合缓存 |
| 小程序审核问题 | 准备隐私政策、用户协议和内容版权说明 |

## 18. 总结

新版设计将小程序拆成两条清晰主线：

- 管理员端：管理题库、章节、题目、导入、考试和数据。
- 用户端：选择题库、选择章节、练习、测试、考试、错题和收藏。

第一版的重点不是复杂商业化，而是让题库从导入到发布、从用户答题到数据统计形成完整闭环。后续再逐步增加会员、支付、AI、数据导出和更复杂的运营能力。
