# 第三方题库小程序设计报告

本报告内容与 `docs/项目设计书.md` 保持一致，作为新版产品方向的独立交付文档。后续若继续实施，应以 `docs/项目设计书.md` 为主文档维护。

## 核心结论

小程序应重构为“登录选择 -> 用户端 / 管理员端”的双入口产品：

- 管理员端负责题库、章节、题目、Word 导入、测试考试配置、用户管理和数据统计。
- 用户端负责题库选择、章节练习、随机测试、考试、错题本、收藏夹和个人记录。
- 普通用户统一称为“用户”，不再拆分其他普通角色。
- 产品定位为第三方题库小程序，文案和业务逻辑不绑定特定组织场景。

完整设计请查看 `docs/项目设计书.md`。
